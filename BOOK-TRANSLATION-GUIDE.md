# 图书翻译格式说明

## 目录结构要求

```
src/content/books/
└── your-book-slug/          # 图书唯一标识（英文、小写、用连字符）
    ├── index.md              # 图书元信息（必需）
    ├── 01-chapter-name.md    # 第1章
    ├── 02-chapter-name.md    # 第2章
    ├── 03-chapter-name.md    # 第3章
    └── ...                   # 更多章节
```

**命名规则：**
- 图书目录：使用英文 slug，如 `deep-learning-basics`、`python-cookbook` 等
- 章节文件：必须以两位数字开头，如 `01-`、`02-`、`10-`、`99-`
- 文件扩展名：`.md`（Markdown 格式）

---

## index.md 格式（图书元信息）

```markdown
---
title: '《书名》中文翻译'
author: '原作者英文名'
translator: '你的名字'
published: 2025-01-12
updated: 2025-01-12
description: '这本书的简介，1-2 句话概括主要内容'
image: ''
tags: ['技术', 'AI', '深度学习']
status: 'ongoing'
draft: false
---

## 译者序

（可选）你对这本书的理解、翻译说明等

## 关于本书

（原书的前言、简介等内容）
```

**字段说明：**
- `title`: 书名，用中文引号包裹
- `author`: 原作者名字
- `translator`: 译者（你的名字）
- `published`: 开始翻译日期（格式：YYYY-MM-DD）
- `updated`: 最后更新日期（格式：YYYY-MM-DD）
- `description`: 简短描述（50-100 字）
- `image`: 封面图路径（暂时留空 `''`）
- `tags`: 标签数组，用于分类
- `status`: 翻译状态
  - `'ongoing'` - 连载中
  - `'completed'` - 已完结
  - `'paused'` - 暂停中
- `draft`: 是否为草稿（`false` 表示公开显示）

---

## 章节文件格式

每个章节一个独立的 `.md` 文件：

```markdown
---
title: '第一章：深度学习基础'
draft: false
---

## 1.1 什么是深度学习

深度学习是机器学习的一个分支...

### 1.1.1 神经网络的历史

神经网络的概念最早可以追溯到...

## 1.2 基本概念

在深入了解之前，我们需要理解几个基本概念：

- **神经元**：神经网络的基本单元
- **激活函数**：引入非线性的关键
- **反向传播**：训练神经网络的核心算法

## 1.3 代码示例

\`\`\`python
import numpy as np

def sigmoid(x):
    return 1 / (1 + np.exp(-x))
\`\`\`
```

**格式要点：**
1. 必须包含 frontmatter（三个短横线之间的部分）
2. `title` 包含章节号和标题
3. `draft: false` 表示公开
4. 正文使用 Markdown 格式
5. 支持代码块、数学公式、图片等

---

## Markdown 格式要求

### 1. 标题层级
- `##` - 章节一级标题（如 1.1）
- `###` - 章节二级标题（如 1.1.1）
- `####` - 章节三级标题（如果需要）

### 2. 代码块
````markdown
```python
# 代码示例
def hello():
    print("Hello, World!")
```
````

支持的语言标识：`python`, `javascript`, `typescript`, `java`, `cpp`, `bash`, `sql` 等

### 3. 数学公式（如需要）
- 行内公式：`$E = mc^2$`
- 块级公式：
  ```markdown
  $$
  f(x) = \int_{-\infty}^{\infty} e^{-x^2} dx
  $$
  ```

### 4. 列表
- 无序列表：使用 `-` 或 `*`
- 有序列表：使用 `1.`, `2.` 等

### 5. 引用
```markdown
> 这是一段引用文本
> 可以跨多行
```

### 6. 强调
- **粗体**：`**文本**`
- *斜体*：`*文本*`
- `代码`：`` `代码` ``

## 注意事项

### ✅ 推荐做法
1. **保持原文结构**：章节划分、标题层级与原书一致
2. **术语一致性**：建立术语表，确保专业术语翻译统一
3. **保留代码**：代码示例保持原样，只翻译注释
4. **及时更新**：定期更新 `updated` 字段
5. **循序渐进**：先翻译前几章测试效果，再继续后续章节

### ❌ 避免错误
1. **不要**修改文件命名格式（必须是 `01-`, `02-`）
2. **不要**遗漏 frontmatter（三个短横线）
3. **不要**在 `title` 字段中使用双引号（用单引号）
4. **不要**在同一目录放置非章节文件（如图片放在单独目录）
5. **不要**使用中文文件名

### 📝 翻译建议
1. **专业术语**：首次出现时给出英文原文，如"神经网络（Neural Network）"
2. **代码注释**：翻译为中文，帮助理解
3. **数学公式**：保持原样，不需要翻译
4. **参考文献**：保留原文，可在末尾添加中文说明
5. **图表标题**：翻译为中文

## 测试清单

翻译完成后，请检查：

- [ ] 目录结构正确（`books/your-book/`）
- [ ] `index.md` 包含完整的 frontmatter
- [ ] 所有章节文件以两位数字开头（`01-`, `02-`）
- [ ] 每个章节文件都有 frontmatter
- [ ] 所有 `title` 字段使用单引号
- [ ] `draft` 设置为 `false`（如果想公开）
- [ ] Markdown 格式正确（标题、列表、代码块）
- [ ] 中英文之间有空格（如"深度学习 Deep Learning"）
- [ ] 专业术语翻译一致

## 快速开始

1. 在 `src/content/books/` 创建新目录，如 `deep-learning-book`
2. 创建 `index.md`，填写图书元信息
3. 创建第一章 `01-introduction.md`
4. 启动开发服务器：`pnpm dev`
5. 访问 `http://localhost:4321/books/` 查看效果
6. 继续翻译后续章节

完成后运行 `pnpm build` 构建生产版本。
